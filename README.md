# 基于FreeRTOS的智能防盗门锁系统设计

---

## 1 课题研究背景与意义
随着家庭安全需求提升，传统机械门锁存在钥匙易丢失、易复制、安全性不足等问题，难以适配智能家居场景。智能门锁融合电子信息、自动控制、无线通信等技术，具备高安全、便捷、可管理、可追溯等优势。

FreeRTOS是开源轻量实时操作系统，适用于嵌入式系统，可实现多任务并发、资源保护与稳定调度。本设计基于**STM32F103ZET6+FreeRTOS**实现智能防盗门锁，具备密码开锁、管理员管理、声光报警、实时时钟、休眠节能等功能，具备工程实践与应用价值。

---

## 2 基本原理与解决方案
系统采用**四层架构**：硬件层→驱动层→操作系统层→应用层。

### 2.1 系统硬件组成
1. **主控模块**：STM32F103ZET6（ARM Cortex-M3，72MHz，512KB Flash/64KB SRAM）
2. **显示模块**：ILI9341 TFT LCD（480×320，FSMC接口）
3. **输入模块**：4×4矩阵键盘（行列扫描，软件消抖）
4. **报警模块**：蜂鸣器+LED（状态声光提示）
5. **存储模块**：AT24C256C EEPROM（掉电保存密码）
6. **时钟模块**：STM32内部RTC（实时时间）
7. **通信模块**：USART（调试/数据传输）
8. **执行模块**：继电器（控制门锁开关）

### 2.2 系统软件架构（FreeRTOS多任务）
1. 初始化任务（InitialTask）：硬件初始化、参数加载、创建任务
2. 按键扫描任务（key_scanTask）：键盘扫描、消抖、按键处理
3. 显示任务（ShowTask）：LCD界面刷新、状态展示
4. 开锁任务（OpenkeyTask）：密码输入、验证、开锁逻辑
5. 管理任务（ManageTask）：管理员模式、密码修改
6. 报警任务（WarnLocktask）：蜂鸣器/LED控制
7. 实时时钟任务（TimeTask）：RTC时间更新与显示
8. 休眠任务（SleepTask）：低功耗休眠与唤醒管理

### 2.3 系统工作流程
1. 待机：显示主界面，等待输入
2. 密码输入：6位密码，*删除、#确认
3. 管理员模式：A进入、B退出，可修改密码
4. 密码验证：正确→继电器开锁；错误→计数+报警
5. 错误锁定：连续3次错误，锁定5秒
6. 输入超时：5秒无操作清空输入
7. 系统休眠：11秒无操作关闭LCD，按键唤醒

## 3 技术方案及开发平台对比论证
### 3.1 主控芯片选择：STM32F103ZET6
- 32位高性能，运算能力强
- 外设丰富（FSMC/RTC/SPI/I2C/USART）
- 开发工具完善（STM32CubeMX、Keil）
- 性价比高，适合嵌入式产品

### 3.2 实时操作系统选择：FreeRTOS
- 开源免费（MIT协议）
- 内核小巧、资源占用低
- 支持任务调度、消息队列、互斥锁、信号量
- 跨平台可移植，社区资源丰富
- 相比裸机：多任务并发、响应更快、资源更安全

### 3.3 关键问题及解决思路
1. **任务间通信**：消息队列异步传输，避免忙等
2. **共享资源保护**：互斥锁保护LCD/EEPROM，防止冲突
3. **按键消抖**：5ms软件延时消抖，确认稳定再处理
4. **数据持久化**：EEPROM存储密码，掉电不丢失
5. **低功耗**：无操作自动休眠，关闭LCD，按键唤醒

## 4 设计实现与测试
### 4.1 系统开发环境
- STM32CubeMX 6.x：硬件配置与代码生成
- Keil MDK 5.x：程序编写、调试、下载
- ST-Link：烧录与在线调试
- Tera Term：串口日志查看

### 4.2 矩阵键盘驱动模块
#### 4.2.1 键盘映射表
```
const uint8_t key_map[4][4] = {
    {'1','2','3','A'},
    {'4','5','6','B'},
    {'7','8','9','C'},
    {'*','0','#','D'}
};
```
- 0-9：数字输入
- *：删除；#：确认
- A：进入管理员；B：退出管理员；C/D：预留

#### 4.2.2 键盘扫描与处理
- 列扫描法检测按键
- 5ms软件消抖
- 按键释放后再处理，避免重复触发
- 每次按键重置休眠计时

### 4.3 开锁任务详细分析
- 每100ms检查是否输入6位密码
- 密码比对：strncmp匹配存储密码
- 正确：继电器吸合4秒，成功提示
- 错误：错误计数+1，报警提示
- 连续3次错误：系统锁定5秒

### 4.4 管理任务详细分析
- A键进入管理员模式
- 输入6位新密码自动保存至EEPROM
- B键退出/修改成功自动退出
- 密码全局变量+闪存同步更新

### 4.5 显示任务详细分析
- 互斥锁保护LCD操作，防止闪烁/乱码
- 支持主界面、开锁成功、密码错误、系统锁定、休眠等界面
- 实时刷新RTC时间
- 区分用户输入/管理员修改界面

### 4.6 报警任务详细分析
支持声光提示类型：
- 开锁成功：长蜂鸣+LED亮
- 密码错误：短蜂鸣3次+LED闪烁
- 输入超时：短蜂鸣
- 系统锁定：LED快速闪烁
- 管理员操作成功：双蜂鸣

### 4.7 休眠任务详细分析
- 5秒无操作且有输入：输入超时，清空缓存
- 11秒无操作：进入休眠，关闭LCD
- 任意按键：重置计时并唤醒系统

### 4.8 程序执行流程
#### 4.8.1 系统启动流程
上电→HAL初始化→时钟配置→外设初始化→FreeRTOS内核启动→初始化任务→加载密码→创建8个任务→删除初始化任务→多任务并发运行

#### 4.8.2 密码开锁流程
按键→扫描消抖→输入缓存→满6位验证→正确开锁/错误报警→锁定保护→返回待机

### 4.9 系统测试与结果分析
#### 4.9.1 功能测试
密码开锁、管理员改密、声光报警、时钟显示、休眠唤醒、错误锁定等功能均正常实现。

#### 4.9.2 性能测试
- 响应时间：按键响应≈50ms，验证≈10ms
- 并发性能：多任务调度正常，无死锁、无冲突
- 功耗：工作≈50mA，休眠≈5mA，降低约90%

## 5 设计总结
### 5.1 任务完成情况
成功实现基于FreeRTOS的智能防盗门锁，完成：
- 矩阵键盘驱动与消抖
- EEPROM密码掉电存储与修改
- TFT LCD多界面显示
- 声光报警提示
- 继电器开锁控制
- RTC实时时钟
- 低功耗休眠与唤醒

### 5.2 实验收获
- 掌握FreeRTOS任务调度、消息队列、互斥锁使用
- 熟练STM32CubeMX+Keil开发流程
- 掌握LCD、EEPROM、键盘、RTC等外设驱动
- 理解嵌入式多任务系统同步与资源保护

### 5.3 问题与解决方案
- LCD闪烁：互斥锁保护显示操作
- 密码验证异常：规范输入缓冲区结束符
- EEPROM写入失败：调整I2C时序
- 系统死锁：合理设置队列与优先级

### 5.4 改进思路
1. 安全性：增加AES加密存储密码
2. 认证方式：添加指纹、IC卡识别
3. 联网功能：集成Wi-Fi/蓝牙，支持手机APP远程控制与日志上传
